HTML Forms

An HTML form is used to collect user input. The user input is most often sent to a server for processing.

The < form > Element

The HTML < form > element is used to create an HTML form for user input

The < form > element is a container for different types of input elements, such as: text fields, checkboxes, radio buttons, submit buttons, etc.

The < input > Element

The HTML <input> element is the most used element of the form. An <input> element can be displayed in many ways, depending on the type attribute.

The Submit Button

The < input type="submit" > defines a button for submitting the form data to a form-handler. The form-handler is typically a file on the server with a script for processing input data. The form-handler is specified in the action attribute of the form.

HTML Form Attributes

Action Attribute

The action attribute defines the action to be performed when the form is submitted. Usually, the form data is sent to a file on the server when the user clicks on the submit button.

Tip: If the action attribute is omitted, the action is set to the current page.

Target Attribute

The target attribute specifies where to display the response that is received after submitting the form.

_blank The response is displayed in a new window or tab.

_self The response is displayed in the current window

_parent The response is displayed in the parent frame

_top The response is displayed in the full body of the window

framename The response is displayed in a named iframe

The default value is _self which means that the response will open in the current window.

Method Attribute

The method attribute specifies the HTTP method to be used when submitting the form data.

The form-data can be sent as URL variables (with method="get") or as HTTP post transaction (with method="post").

The default HTTP method when submitting form data is GET.

Use of Get

Appends the form data to the URL, in name/value pairs. NEVER use GET to send sensitive data! (the submitted form data is visible in the URL!). The length of a URL is limited (2048 characters). Useful for form submissions where a user wants to bookmark the result. GET is good for non-secure data, like query strings in Google.



Use of Post

Appends the form data inside the body of the HTTP request (the submitted form data is not shown in the URL) POST has no size limitations, and can be used to send large amounts of data. Form submissions with POST cannot be bookmarked



Tip: Always use POST if the form data contains sensitive or personal information!

Autocomplete Attribute

The autocomplete attribute specifies whether a form should have autocomplete on or off.

Novalidate Attribute

The novalidate attribute is a boolean attribute.

When present, it specifies that the form-data (input) should not be validated when submitted.

Default Behavior : Normally, HTML5 forms include built-in validation for input fields , such as checking if required fields are filled, email fields contain valid email addresses, etc. If any validation rule fails, the form cannot be submitted, and the browser displays error messages.

With novalidate: Adding this attribute prevents the browser from performing client-side validation. The form can be submitted even if the fields are invalid, leaving validation to be handled manually using JavaScript or server-side processing.

Use Cases: When you want custom validation logic instead of relying on the browser's default validation. For scenarios where server-side validation is preferred for security reasons. During development and testing phases to bypass validation temporarily.

The Name Attribute for <input>

Notice that each input field must have a name attribute to be submitted. If the name attribute is omitted, the value of the input field will not be sent at all.

HTML Input Attributes

No 1 : The value Attribute

The input value attribute specifies an initial value for an input field.

No 2 : readonly Attribute

The input readonly attribute specifies that an input field is read-only.

No 3 : disabled Attribute

The input disabled attribute specifies that an input field should be disabled.

No 4 : size Attribute

The input size attribute specifies the visible width, in characters, of an input field. The default value for size is 20.

No 5 : maxlength Attribute

The input maxlength attribute specifies the maximum number of characters allowed in an input field.

No 6 : min and max Attributes

The input min and max attributes specify the minimum and maximum values for an input field.

No 7 : multiple Attribute

The input multiple attribute specifies that the user is allowed to enter more than one value in an input field.

No 8 : pattern Attribute

The input pattern attribute specifies a regular expression that the input field's value is checked against, when the form is submitted.

If the input value does not match the pattern, the form cannot be submitted.

To display an error message, use the title attribute along with pattern.

The + in the regular expression is a quantifier that means: Match one or more occurrences of the preceding character or group.

NO 9 : placeholder Attribute

The input placeholder attribute specifies a short hint that describes the expected value of an input field (a sample value or a short description of the expected format).

No 10 : required Attribute

The input required attribute specifies that an input field must be filled out before submitting the form.

No 11 : step Attribute

The input step attribute specifies the incremental value for valid inputs. . Example: if step="3", legal numbers could be ...-6,-3, 0, 3, 6,.. etc.

No 12 : autofocus Attribute

The input autofocus attribute specifies that an input field should automatically get focus when the page loads.

No 13 : height and width Attributes

The input height and width attributes specify the height and width of an input type ="image" element.

No 14 : list Attribute

The input list attribute refers to a datalist element that contains pre-defined options for an input element.

No 15 : autocomplete Attribute

The input autocomplete attribute specifies whether a form or an input field should have autocomplete on or off.

Question : is novalidate and formnovalidate same

Answer : No, novalidate and formnovalidate are not the same, but they serve related purposes in HTML forms.

1. novalidate Attribute (On form element):

Usage : Applied to the form element.

Purpose : It disables all form validation for the entire form.

Effect : When novalidate is used on a form, the browser will not automatically validate the form's input fields before submitting. This means it won't check for required fields, pattern matching, or any other validation constraints.

2. formnovalidate Attribute (On button or input type="submit"):

Usage : Usage: Applied to individual elements, like submit buttons etc

Purpose : It disables validation only for the specific button it is applied to when that button is used to submit the form.

Effect : When formnovalidate is used, the form will skip validation only when that specific button is clicked, but other submit buttons will still trigger validation.

Data Validation

Data validation is the process of ensuring that user input is clean, correct, and useful.

Validation can be defined by many different methods, and deployed in many different ways.

Client side validation is performed by a web browser, before input is sent to a web server.

Server side validation is performed by a web server, after input has been sent to the server.

HTML Constraint (zabrdasti, pabandi) Validation

HTML constraint validation is based on:

  • Constraint validation HTML Input Attributes
  • Constraint validation CSS Pseudo Selectors
  • Constraint validation DOM Properties and Methods

Constraint Validation HTML Input Attributes

Attribute Description

Constraint Validation CSS Pseudo Selectors

Selector Description
:disabled Selects input elements with the "disabled" attribute specified
:invalid Selects input elements with invalid values
:optional Selects input elements with no "required" attribute specified
:required Selects input elements with the "required" attribute specified
:valid Selects input elements with valid values
